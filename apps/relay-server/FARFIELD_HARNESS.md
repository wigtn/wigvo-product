# Session A far-field 하네스

전화를 걸지 않고 far-field(마이크 거리·잔향·사무실 소음) 조건에서 Session A 입력
파이프라인을 검증한다. **실제 Whisper 전사**까지 태우므로 VAD 대리지표가 아니라
전사 품질(CER)·발화 누락·할루시를 직접 측정한다.

```bash
cd apps/relay-server
OPENAI_API_KEY=... uv run python scripts/farfield_harness.py
```

## 재현하는 경로
```
합성 조건 오디오 → [far-field 체인] → ClientVAD 커밋 → 서버 에너지 게이트(#86)
                → 살아남은 세그먼트만 Whisper 전사 → 정답 대조
```
- 발화: macOS `say`(Yuna) 문장 3개 + 사이 무음 1.5s (정답 구간을 알고 시작)
- 조건: 거리 감쇠 × 합성 RIR(RT60 0.4s) × **다화자 babble**(핑크노이즈와 달리 STT
  오염을 실제로 유발하는 노이즈 유형 — 목소리 4개를 오프셋 겹침)
- 체인 3-way: `OFF`(파필드 없음) / `ON`(디노이즈+적응형VAD+AGC) / `ON+게이트`(게이트형 AGC)

## 측정 결과 (2026-07-19)

| 조건 | OFF | ON | **ON+게이트(현행)** |
|---|---|---|---|
| 근접·조용 | 3/3 · CER 13.2 | 3/3 · **9.9** | 3/3 · 13.2 |
| 1.5m·잔향·babble약 | 3/3 · 10.8 | 2커밋 병합 · **69.3** | 3/3 · **8.8** |
| 1.5m·잔향·babble강 | 1커밋 · 216 | 1커밋 · 203 | 1커밋 · 214 |
| **3m·잔향·babble약** | **0/3 (전멸)** | 3/3 · 220 | **3/3 · 17.1** |

읽는 법:
1. **far-field 체인의 본체 가치는 3m** — 없으면 발화가 하나도 커밋되지 않는다(0/3).
2. **AGC를 speech-gate 없이 쓰면 역효과** — 무음 구간에서 게인이 최대로 붙어 babble을
   발화 수준으로 키우고, VAD가 무음을 못 봐 세그먼트가 병합된다 → 옆사람 말이 함께
   전사(CER 69/220). 게이트형으로 고치면 8.8/17.1.
3. **SNR ~3dB(babble강)는 세 체인 모두 실패** — 이 수준의 소음은 프론트엔드 전처리로
   풀리지 않는다(마이크·환경 개선 또는 서버측 화자분리 영역).

## 한계 (실통화 측정으로 대체 불가)
- 합성 RIR·TTS 발화·TTS babble이며 실제 녹음 데이터가 아니다.
- 브라우저 기본 `noiseSuppression`이 모델링되지 않아 실제로 체인에 도달하는 소음은
  더 약하다 → babble 조건은 보수적(비관적) 추정.
- Speex 디노이저 대신 스펙트럴 게이팅 프록시를 사용한다.
