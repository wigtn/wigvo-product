# Session A far-field 평가

전화를 걸지 않고 far-field(마이크 거리·잔향·소음) 조건에서 Session A 입력 파이프라인을
검증한다. 세 가지 데이터셋을 쓰며, **결론은 실측 데이터셋 기준으로 적는다.**

| 스크립트 | 데이터 | 기준 전사 |
|---|---|---|
| `scripts/farfield_harness.py` | 한국어 TTS + 합성 RIR + TTS babble (합성) | 원문 텍스트 |
| `scripts/ami_farfield_eval.py` | **AMI Meeting Corpus** — 같은 발화를 헤드셋/원거리 마이크로 동시 녹음 | 헤드셋 채널 Whisper 전사 |
| `scripts/librispeech_farfield_eval.py` | **LibriSpeech + 실측 RIR + 실제 소음**(OpenSLR RIRS_NOISES) | **공식 사람 전사** |

```bash
cd apps/relay-server
OPENAI_API_KEY=... uv run python scripts/farfield_harness.py
AMI_DIR=... OPENAI_API_KEY=... uv run python scripts/ami_farfield_eval.py
LIBRI_DIR=... RIRS_DIR=... OPENAI_API_KEY=... uv run python scripts/librispeech_farfield_eval.py
```

## 결론 (2026-07-19)

**far-field 체인(디노이즈 + 적응형 VAD + AGC)의 전사 정확도 개선은 실측 데이터에서
확인되지 않았다.** 큰 개선을 보인 것은 합성 하네스뿐이며, 그 하네스는 우리 체인에
유리하게 편향돼 있었다(아래 "합성이 과대평가된 이유").

체인을 유지하는 근거는 **발화 유실 방지**(AMI 미검출 5→1)와 근접·조용 조건에서
해가 없다는 점이며, "far-field에서 정확도가 크게 오른다"는 주장은 **철회한다.**

### 1. AMI — 실제 회의실, near/far 동시 녹음 (n=150, 회의 3개 × 화자 4명)

| 체인 | WER% 평균 | 중앙값 | 미검출 |
|---|---|---|---|
| OFF (파필드 없음) | 22.3 | 11.8 | 5 |
| ON (#1 원안) | 21.3 | 10.5 | 4 |
| ON+게이트 (현행) | 23.9 | 12.5 | **1** |

WER은 세 팔이 오차 범위 내. 유일하게 일관된 이득은 **미검출**(VAD가 끝내 커밋하지
않아 통화에서 통째로 유실될 발화)이 5 → 1로 줄어든 것.

### 2. LibriSpeech + 실측 RIR + 실제 소음 (n=120, 공식 정답 전사, 표준 SNR 스윕)

| SNR | OFF | ON (#1) | ON+게이트 |
|---|---|---|---|
| 20 dB | 40.0 (중앙 16.0) | 39.1 (16.0) | 39.9 (15.4) |
| 10 dB | 44.4 (23.6) | 47.3 (23.6) | 45.0 (25.0) |
| 5 dB | 49.6 (42.9) | 48.5 (42.1) | 58.9 (42.8) |

전 SNR에서 차이 없음. 5 dB의 ON+게이트 평균 58.9는 소수 이상치가 끌어올린 값이며
중앙값은 42.8로 다른 팔과 같다.

### 3. 한국어 합성 하네스 (n=30) — 참고용, 결론 근거로 쓰지 말 것

| 조건 | OFF | ON (#1) | ON+게이트 |
|---|---|---|---|
| 근접·조용 | 30/30 · CER 6.3 | 30/30 · 5.6 | 30/30 · 6.5 |
| 1.5m·잔향·babble약 | 30/30 · 7.6 | 25커밋 병합 · 52.7 | 30/30 · 8.3 |
| 1.5m·잔향·babble강 | 1커밋 · 2986 | 2992 | 2992 (셋 다 실패) |
| 3m·잔향·babble약 | 9/30 · 35.6 | 29/30 · 70.4 | 30/30 · 12.0 |

여기서 보이는 큰 개선(3m CER 35.6 → 12.0)은 **실측 두 데이터셋에서 재현되지 않았다.**

> 이전 판의 "3m에서 OFF는 발화 0/3 전멸" 수치는 n=3 표본의 우연이었다. n=30에서는
> 9/30이며, 실측 데이터에서는 해당 격차 자체가 나타나지 않는다.

## 합성이 과대평가된 이유

합성 조건에서 "거리"를 **단순 음량 감쇠(스칼라 곱)** 로 모델링했다. 그 열화는 AGC가
곱셈 한 번으로 정확히 되돌릴 수 있는 형태이므로 체인이 실제보다 훨씬 잘 듣는 것처럼
보인다. 실제 far-field 열화는 잔향·스펙트럼 왜곡·SNR 저하가 섞여 있어 게인으로 복구되지
않는다. 합성 하네스는 **회귀 감지용**으로만 쓰고, 성능 주장은 AMI/LibriSpeech로 한다.

## 알려진 측정 한계

- `librispeech_farfield_eval.py`는 살아남은 커밋 구간을 합쳐 한 번에 전사하므로
  **검출 실패가 마스킹된다**(발화유실이 항상 0으로 나옴). 지금 유일하게 이득이 보이는
  축이 검출인데 이 스크립트로는 재지 못한다 — FRR 중심 평가가 후속 과제.
- AMI 기준 전사는 헤드셋 채널의 Whisper 출력이라 완전한 정답이 아니다(수동 전사 미사용).
- AMI·LibriSpeech 모두 영어다. 한국어 far-field 공개 데이터가 사실상 없어 언어 정합성은
  합성 하네스로만 확인된다.
- 브라우저 기본 `noiseSuppression`(프로덕션에서 항상 ON)은 어느 평가에도 모델링돼 있지 않다.
- 실통화 검증은 여전히 미실시.
