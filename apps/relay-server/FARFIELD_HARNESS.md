# Session A far-field 평가

전화를 걸지 않고 far-field(마이크 거리·잔향·소음) 조건에서 Session A 입력 파이프라인을
검증한다. 세 가지 데이터셋을 쓰며, **결론은 실측 데이터셋 기준으로 적는다.**

| 스크립트 | 데이터 | 기준 전사 |
|---|---|---|
| `scripts/farfield_harness.py` | 한국어 TTS + 합성 RIR + TTS babble (합성) | 원문 텍스트 |
| `scripts/ami_farfield_eval.py` | **AMI Meeting Corpus** — 같은 발화를 헤드셋/원거리 마이크로 동시 녹음 | 헤드셋 채널 Whisper 전사 |
| `scripts/librispeech_farfield_eval.py` | **LibriSpeech + 실측 RIR + 실제 소음**(OpenSLR RIRS_NOISES) | **공식 사람 전사** |
| `scripts/detection_eval.py` | 위 두 코퍼스 — **FRR/FAR 표준 VAD 지표** (Whisper 불필요) | 발화 구간 라벨 |
| `scripts/differential_check.py` | 팔끼리 갈린 발화만 전사해 **복구분의 실사용 가치** 확인 | 헤드셋 채널 |

```bash
cd apps/relay-server
OPENAI_API_KEY=... uv run python scripts/farfield_harness.py
AMI_DIR=... OPENAI_API_KEY=... uv run python scripts/ami_farfield_eval.py
LIBRI_DIR=... RIRS_DIR=... OPENAI_API_KEY=... uv run python scripts/librispeech_farfield_eval.py

# 검출률(FRR/FAR) — API 호출 없음, 표본을 크게 잡을 수 있다
LIBRI_DIR=... RIRS_DIR=... uv run python scripts/detection_eval.py
AMI_DIR=... uv run python scripts/detection_eval.py --ami

# 차등 검증 — 팔끼리 갈린 발화만 전사
AMI_DIR=... OPENAI_API_KEY=... uv run python scripts/differential_check.py --ami
```

## 결론 (2026-07-19)

**far-field 체인(디노이즈 + 적응형 VAD + AGC)의 전사 정확도(WER) 개선은 실측 두
코퍼스에서 확인되지 않았다.** 검출률(FRR)도 데이터셋 간 일관되지 않는다 — AMI에서는
개선되지만 LibriSpeech에서는 재현되지 않는다(오히려 소폭 높음).

확인된 실익은 좁다:
- **AMI형 조건**(연속 대화·다화자 회의실)에서 통째로 유실될 발화 **9건을 복구, 역행 0건**.
  복구된 내용은 실제로 전사 가능했다(WER 31.8%, 코퍼스 평균 22%).
- 게이트는 **저 SNR에서 FAR를 낮춘다**(10 dB 59.2 → 51.0, 5 dB 66.9 → 59.4).
- 근접·조용 조건에서 무해.

"far-field에서 정확도가 크게 오른다"는 주장은 **철회한다.**

### 1. AMI — 실제 회의실, near/far 동시 녹음 (n=150, 회의 3개 × 화자 4명)

| 체인 | WER% 평균 | 중앙값 | 미검출 |
|---|---|---|---|
| OFF (파필드 없음) | 22.3 | 11.8 | 5 |
| ON (#1 원안) | 21.3 | 10.5 | 4 |
| ON+게이트 (현행) | 23.9 | 12.5 | **1** |

WER은 세 팔이 오차 범위 내. 유일하게 일관된 이득은 **미검출**(VAD가 끝내 커밋하지
않아 통화에서 통째로 유실될 발화)이 5 → 1로 줄어든 것.

### 2. LibriSpeech + 실측 RIR + 실제 소음 (n=120, 공식 정답 전사, 표준 SNR 스윕)

| SNR | OFF | ON (#1) | ON+게이트 |
|---|---|---|---|
| 20 dB | 40.0 (중앙 16.0) | 39.1 (16.0) | 39.9 (15.4) |
| 10 dB | 44.4 (23.6) | 47.3 (23.6) | 45.0 (25.0) |
| 5 dB | 49.6 (42.9) | 48.5 (42.1) | 58.9 (42.8) |

전 SNR에서 차이 없음. 5 dB의 ON+게이트 평균 58.9는 소수 이상치가 끌어올린 값이며
중앙값은 42.8로 다른 팔과 같다.

### 3. 한국어 합성 하네스 (n=30) — 참고용, 결론 근거로 쓰지 말 것

| 조건 | OFF | ON (#1) | ON+게이트 |
|---|---|---|---|
| 근접·조용 | 30/30 · CER 6.3 | 30/30 · 5.6 | 30/30 · 6.5 |
| 1.5m·잔향·babble약 | 30/30 · 7.6 | 25커밋 병합 · 52.7 | 30/30 · 8.3 |
| 1.5m·잔향·babble강 | 1커밋 · 2986 | 2992 | 2992 (셋 다 실패) |
| 3m·잔향·babble약 | 9/30 · 35.6 | 29/30 · 70.4 | 30/30 · 12.0 |

여기서 보이는 큰 개선(3m CER 35.6 → 12.0)은 **실측 두 데이터셋에서 재현되지 않았다.**

> 이전 판의 "3m에서 OFF는 발화 0/3 전멸" 수치는 n=3 표본의 우연이었다. n=30에서는
> 9/30이며, 실측 데이터에서는 해당 격차 자체가 나타나지 않는다.

### 4. 검출률 — FRR/FAR (`detection_eval.py`, Whisper 불필요)

WER 평가로는 "VAD가 아예 커밋하지 않은 발화"를 잴 수 없다(전송되지 않은 것은 전사
지표에 나타나지 않는다). VAD 레벨에서 표준 지표를 직접 측정한다.

**AMI (FRR %, 원거리 마이크)**

| 회의 (발화 수) | OFF | ON (원안) | ON+게이트 |
|---|---|---|---|
| ES2002a (70) | 9.2 | 7.4 | **7.5** |
| ES2002b (254) | 6.8 | 5.1 | **3.1** |
| ES2003a (61) | 6.8 | 11.3 | **5.2** |

AMI의 FAR는 보고에서 제외한다 — 정답 라벨(헤드셋 에너지 VAD)이 실제 발화를 상당수
놓쳐 "비발화"로 잘못 표시하므로 값이 신뢰할 수 없다(50~75%로 나온다).

**LibriSpeech + 실측 RIR (FRR / FAR %, n=200, 정답 정확)**

| SNR | OFF | ON (원안) | ON+게이트 |
|---|---|---|---|
| 20 dB | 11.3 / 15.4 | 5.1 / 32.0 | 12.9 / 20.0 |
| 15 dB | 8.6 / 36.6 | 4.2 / 44.4 | 8.4 / 37.8 |
| 10 dB | 3.8 / 59.2 | 1.8 / 56.4 | 5.5 / **51.0** |
| 5 dB | 3.0 / 66.9 | 1.7 / 63.2 | 3.9 / **59.4** |

**AMI의 FRR 개선이 여기서는 재현되지 않는다.** 게이트의 효과는 FRR이 아니라 저 SNR에서의
FAR 감소로 나타난다. 게이트 없는 원안은 FRR이 가장 낮지만 20/15 dB에서 FAR가 급증
(15.4 → 32.0) — 무음까지 증폭해 소음을 발화로 만드는 실패 모드다.

FAR 절대값이 전 구간 15~67%로 높다는 점도 기록해 둔다. 비발화 구간이 STT로 흘러가는
양이 많다는 뜻이며, 할루시 경로가 열려 있다.

### 5. 차등 검증 (`differential_check.py`)

검출률만으로는 부족하다 — 더 많이 커밋했는데 내용이 소음이면 FRR은 좋아지고 사용자
경험은 나빠진다. 팔끼리 결과가 갈린 발화만 실제로 전사해 확인했다.

| 구분 | 건수 | 복구분 전사 WER |
|---|---|---|
| 현행만 검출 (OFF 유실) — 복구 | **9** | 31.8 % (중앙값 30.0) |
| OFF만 검출 (현행 유실) — 역행 | **0** | — |

복구분은 코퍼스 평균(22%)보다 나쁘지만 내용이 전달되는 수준이며 "잡았지만 쓸모없음"은
아니다. LibriSpeech 조건에서는 두 팔 모두 발화를 통째로 놓치는 일이 없어(완전유실 0%)
차등 대상이 0건이었다.

## 합성이 과대평가된 이유

합성 조건에서 "거리"를 **단순 음량 감쇠(스칼라 곱)** 로 모델링했다. 그 열화는 AGC가
곱셈 한 번으로 정확히 되돌릴 수 있는 형태이므로 체인이 실제보다 훨씬 잘 듣는 것처럼
보인다. 실제 far-field 열화는 잔향·스펙트럼 왜곡·SNR 저하가 섞여 있어 게인으로 복구되지
않는다. 합성 하네스는 **회귀 감지용**으로만 쓰고, 성능 주장은 AMI/LibriSpeech로 한다.

## 알려진 측정 한계

- `librispeech_farfield_eval.py`는 살아남은 커밋 구간을 합쳐 한 번에 전사하므로
  검출 실패가 마스킹된다 — 그래서 `detection_eval.py`를 별도로 둔다(WER 표는 여전히
  이 한계를 갖는다).
- AMI의 FAR는 정답 라벨 부실로 신뢰할 수 없어 FRR만 사용한다.
- 차등 검증 대상이 9건으로 표본이 작다.
- AMI 기준 전사는 헤드셋 채널의 Whisper 출력이라 완전한 정답이 아니다(수동 전사 미사용).
- AMI·LibriSpeech 모두 영어다. 한국어 far-field 공개 데이터가 사실상 없어 언어 정합성은
  합성 하네스로만 확인된다.
- 브라우저 기본 `noiseSuppression`(프로덕션에서 항상 ON)은 어느 평가에도 모델링돼 있지 않다.
- 실통화 검증은 여전히 미실시.
